-----
### 퀴즈
-----
1. 카프카는 메모리에 데이터를 저장하기 때문에 장애가 났을 경우 데이터가 유실 된다. (X) 
   - 프로듀서에서 데이터를 보내게 되면, 카프카 브로커에 데이터 저장
   - 브로커 프로세스에 내부적으로 파일 시스템에 데이터를 저장 (.log 형식)
   - 따라서, 장애가 나더라도 유실되지 않음
     
2. 카파 아키텍처는 람다 아키텍처를 개선한 데이터 아키텍처이다. (O)

3. 서빙 레이어는 데이터를 프로세싱하는데 사용되는 곳이다. (X)
   - 서빙 레이어 : 기존 스피드 레이어 혹은 람다 아키텍쳐에서 배치 레이어에서 데이터 처리를 수행한 뒤 데이터를 서빙 및 활용하는 용도로 사용

-----
### 정리
-----
1. 아파치 카프카 : Linked-In에서 내부 아키텍쳐를 개선하기 위해 고안한 플랫폼
2. 카프카는 소스 애플리케이션과 타깃 애플리케이션 간의 커플링을 제거하기 위해 중앙 집중화 구조를 가짐
3. 카프카의 파티션 내부 구조는 메세지 큐와 동일한 FIFO 구조
4. 카프카는 높은 처리량, 확장성, 영속성, 고가용성 특징을 가짐
5. 람다 아키텍쳐에서 카프카는 스피드 레이어에 위치
   - 람다 아키텍쳐는 프로세싱 레이어가 2개로 나뉘어지므로 로직이 2개로 나뉘어진다는 단점 존재
6. 카파 아키텍쳐는 람다 아키텍쳐의 단점을 개선한 아키텍쳐
7. 스트리밍 데이터 레이크는 서빙 레이어를 제거하고 카프카를 Long-Term 저장소로 사용하는 것
