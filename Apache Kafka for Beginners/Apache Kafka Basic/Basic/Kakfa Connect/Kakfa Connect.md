-----
### 카프카 커넥트 (Kakfa Connect)
-----
1. 카프카에서 데이터 파이프라인을 반복적으로 만들어내고 개발하고 운영할 때 효과적
2. 카프카에서 공식적으로 제공하는 컴포넌트 중 하나
   - 커넥트와 커넥터로 구성
   - 커넥트(Connect) : 커넥터가 동작하도록 실행해주는 프로세스로, 파이프라인으로 동작하는 커넥터를 동작하기 위해 반드시 커넥트를 실행해야 함
   - 커넥터(Connector) : 실질적으로 데이터를 처리하는 코드가 담긴 jar 패키지를 의미 (즉, 특정 동작을 하는 코드 뭉치)

3. 커넥터
   - 싱크 커넥터(Sink Connector) : 특정 토픽에 있는 데이터를 Oracle, MySQL, ES와 같이 특정 저장소에 저장하는 역할 (컨슈머와 같은 역할)
   - 소스 커넥터(Source Connector) : 데이터베이스로부터 데이터를 가져와 토픽에 넣는 역할 (프로듀서 역할)

4. 카프카 커넥트
   - 단일 실행 모드 커넥트 : 간단한 데이터 파이프라인을 구성하거나 개발용으로 주로 사용 
   - 분산 모드 커넥트 : 여러 개의 프로세스를 한 개의 클러스터에 묶어 사용 (즉, 2개 이상의 커넥터를 클러스터로 묶는 것), 일부 커넥트에 장애가 발생하더라도 파이프라인을 자연스럽게 FailOver해서 나머지 실행중인 커넥트에서 데이터를 지속적 처리할 수 있도록 도와줌

5. 동작 방식
   - 커넥트를 실행할 때, 커넥터가 어디에 위치하는지 config 파일에 위치를 지정해야 함 (plugin.path=/var/connectors : 커넥터 jar 패키지에 있는 디렉토리를 config 파일에 지정)
   - 커넥트를 실행하면, jar 파일의 커넥터들을 함께 모아서 커넥트를 실행할 수 있도록 준비 상태에 돌입
   - 실행중인 커넥트에서 커넥터를 사용하려면 REST API를 통해 커넥터 실행 가능
   - 파이프라인을 만들 때 추가 개발 없이 배포를 하는 과정 없이 REST API를 통해 커넥터를 통한 파이프라인이 분산해서 생기는 것
