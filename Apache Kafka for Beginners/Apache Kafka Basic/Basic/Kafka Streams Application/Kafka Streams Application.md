### 카프카 스트림즈 애플리케이션 (Kafka Streams Application)
-----
1. 카프카에서 공식적으로 제공하는 자바 라이브러리 (JVM 기반)
2. 토픽에 있는 데이터를 낮은 지연과 함께 빠른 속도로 데이터를 처리 가능
3. 장점
   - 카프카와 완벽 호환되며, 성능개선도 빠르게 이루어짐 (무엇보다도 유실이나 중복 처리 되지 않고, 딱 한 번만 처리될 수 있는 강력한 기능을 가지고 있음)
   - 스케줄링 도구가 필요 없음
   - 스트림즈 DSL과 프로세서 API를 제공 : 대부분의 스트림즈 기능은 스트림즈 DSL로 구현, 부가적 추가 구현 기능은 프로세서 API를 통해 직접 구현 가능
   - 로컬 상태 저장소를 사용
     + 비상태기반(Stateless) 처리 : 필터링이나 데이터를 변환하는 처리로, 데이터가 들어오는 즉시 처리하고 프로듀스하므로 유실이나 중복 우려가 적어 쉽게 개발
     + 상태기반(Stateful) 처리 : 윈도우(Window), 조인(Join), 취합(Aggregation) 등 이전에 받았던 데이터를 프로듀스가 메모리에 저장하면서 다음 데이터를 참조해서 처리해야 함
     + 상태기반 분산 프로세스 처리 어려움을 도와주는 것이 스트림즈로, 로컬에 RocksDB를 사용해 상태를 저장하고, 상태에 대한 변환 정보는 카프카의 변경 로그(ChangeLog) 토픽에 저장하여, 프로세서에 장애가 발생하더라도, 그 상태가 모두 안전하게 저장되므로 자연스럽게 장애 복구가 가능
