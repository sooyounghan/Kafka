-----
### 아파치 카프카의 미래
-----
1. 카프카는 2011년 Linked-In에서 개발되어 2011년 오픈소스로 공개
   - 데이터 처리를 여러 애플리케이션에서 하는 것이 아닌 중앙에서 처리할 수 있도록 중앙집중화한 것
   - 즉, 데이터 스트림을 한 곳에서 실시간으로 관리가 가능하게 하기 위한 목적으로 개발

2. 기업의 대용량 데이터를 수집하고, 사용자들이 실시간 스트림으로 소비할 수 있게 만들어주는 역할
3. 카프카의 특징
   - High Throughput Message Capacity : 짧은 시간 내 엄청난 양의 데이터를 컨슈머에게 전달 가능하며, 파티션을 통한 분산처리가 가능하므로 데이터 양이 많아질수록 컨슈머 개수를 늘려서 병렬 처리가 가능하고 이를 통해 데이터 처리를 더욱 빠르게 할 수 있음
   - Scalability와 Fault Tolerant : 카프카는 확장성이 뛰어남 (이미 사용되고 있는 카프카 브로커가 있다고 하더라도 신규 브로커 서버를 추가해 수평 확장이 가능하며, 늘어난 브로커 중 일부가 죽더라도, 일부 Replica로 복제된 데이터는 안전하게 보관되어 있으므로 복구하여 처리 가능)
   - Undeleted Log : 다른 플랫폼과 달리 컨슈머가 데이터를 가지고 가더라도 데이터가 사라지지 않음 (카프카에서는 컨슈머의 그룹 아이디만 다르다면 동일한 데이터도 각각 다른 형태로 처리 가능)
